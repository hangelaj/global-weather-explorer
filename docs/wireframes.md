# Wireframes – Global Weather Explorer

## Introduction

Wireframes were created during the planning stage of the Global Weather Explorer project.
They represent the basic layout and structure of the user interface before development began.
The goal of the wireframes is to visualise how users will interact with the application and where key elements will be positioned.

The design focuses on simplicity, clarity, and user-friendly navigation.

---

## Homepage Layout

The main interface of the application includes the following sections:

1. Header
2. Weekly calendar display
3. Weather search section
4. Current weather display
5. Five-day weather forecast
6. Footer

---

## Wireframe Structure

### Header Section

Purpose:
Displays the project title and introduces the application.

Elements:

* Application title: Global Weather Explorer
* Short description
* Weekly calendar showing days from Sunday to Saturday

Example layout:

[ Global Weather Explorer ]

Explore real-time weather conditions around the world

Sun  Mon  Tue  Wed  Thu  Fri  Sat

---

### Search Section

Purpose:
Allows the user to search for a city's weather.

Elements:

* Input box for city name
* Search button
* Error message area

Example layout:

Search Weather

[ Enter City Name ]  [ Search ]

(Error message if city not found)

---

### Current Weather Section

Purpose:
Displays the current weather information for the searched city.

Elements:

* City logo/image
* City name
* Weather icon
* Temperature
* Weather description
* Humidity
* Wind speed

Example layout:

City Logo

City Name

Temperature
Weather Description
Humidity
Wind Speed

---

### Forecast Section

Purpose:
Shows a five-day weather forecast.

Each forecast card displays:

* Date
* Weather icon
* Temperature
* Weather condition

Example layout:

[Day 1]   [Day 2]   [Day 3]   [Day 4]   [Day 5]

---

### Footer

Purpose:
Provides copyright information and attribution for the weather API.

Example layout:

© 2026 Global Weather Explorer
Weather data provided by OpenWeather API

---

## Design Considerations

The following principles were used when designing the wireframes:

Simplicity
The interface is kept clean and uncluttered so users can quickly search for weather information.

Responsiveness
The layout is designed to work on both desktop and mobile devices.

Accessibility
Clear text, spacing, and color contrast help improve readability and usability.

---

## Future Improvements

Possible improvements to the wireframe design include:

* Adding a map to display the searched location
* Allowing users to save favourite cities
* Adding weather animations or background themes
